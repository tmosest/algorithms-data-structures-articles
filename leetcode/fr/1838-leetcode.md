Problème 1838 – Fréquence de l'élément le plus fréquent
** Difficulté :** Moyenne **LeetCode URL:** https://leetcode.com/problèmes/fréquence-de-l'élément le plus fréquent/

> **Objectif:**
> Dans une question d'interview à passe unique, vous êtes demandé de maximiser la fréquence d'un élément que vous pouvez obtenir en exécutant *au maximum* des opérations d'accroissement `k` sur un tableau entier `nums`.
> (Une opération d'accroissement signifie ajouter 1 à n'importe quel élément du tableau. La valeur de `k` vous indique combien de fois vous pouvez faire cela.)

Aperçu du problème

Texte
Entrée: nombres = [1,2,4,5], k = 5
Produit : 4
«» "

Vous pouvez ajouter au plus `k` à tous les éléments. Après les opérations, quelle est la fréquence maximale ** d'une valeur unique dans le tableau?

---

L'approche – La partie « Bonne »

1. **Trier le tableau**
*Pourquoi?* Trier ensemble des groupes des valeurs égales ou proches. Une fois trié, la seule chose qui compte est le nombre d'éléments que vous pouvez élever à une valeur cible (l'élément le plus droit de la fenêtre courante).

2. **Fenêtre coulissante sur le tableau trié**
* Maintenez une fenêtre `[gauche ... à droite]` *
* `total` = somme de tous les éléments à l`intérieur de la fenêtre
* État:
Texte
nums[right] * windowTaille <= total + k
«» "
Si le côté gauche est violé, réduire la fenêtre de la gauche jusqu'à ce que l'état soit à nouveau satisfait.

3. **Track la meilleure taille de fenêtre** – cette taille est la réponse.

> **Pourquoi ça marche* *
> Dans une fenêtre triée, la *cible* est toujours la valeur la plus droite. Toutes les autres valeurs de la fenêtre doivent être augmentées à cette cible.
> Le nombre d'augmentations nécessaires est
> ```texte
> cible * fenêtre Taille – total
> `` "
> Si ce nombre est ≤ `k`, la fenêtre est faisable; sinon nous devons laisser tomber des éléments de gauche.

## Le "Bad" – Edge-Cas & Gotchas

Question Comment l'éviter
-- -- -- -- -- -- -- --
**Débordement entier** – `nums[right] * windowSize` peut dépasser la plage `int`. Autres
** Grandes tailles d'entrée** – Le "total" peut également devenir important. Calculer «total» dans un «long long»/«long». Autres
Autres ** État de boucle difficile** – Oublier de soustraire `nums[left]` lorsque le rétrécissement peut conduire à de mauvaises réponses. Suivez attentivement la boucle interne et mettez toujours à jour `total` et `gauche`. Autres

Les erreurs courantes

Pourquoi ça arrive ?
- Oui.
**Treating the array as a multiset of frequency**= Beaucoup de gens pensent que l'augmentation des plus petits nombres est la réponse. Ce serait O(n k) et échoue sur les plus grands tests. Autres Réaliser la fenêtre doit cibler le nombre *le plus grand* dans la fenêtre (`nums[right]`). Autres
La pensée « k » est une limite par élément au lieu d'un budget global. Utilisez `total + k` comme somme *disponible* pour la fenêtre actuelle. Autres
**Au lieu de `à droite - à gauche + 1`. Autres
**Pas de tri**= Oublier que la logique repose sur l'ordre trié. Ajouter `sort(nums.begin(), nums.end()` (Java: `Arrays.sort(nums)`). Autres

La mise en œuvre – Solution d'une ligne en trois langues

Ci-dessous vous trouverez une mise en œuvre prête à la production, entièrement commentée dans **Java, Python et C++**. Tous les trois utilisent le même algorithme d'espace O(n log n) time + O(1) décrit ci-dessus.

C'est pas vrai. Java

"Java
Importer java.util. Les tableaux;

solution de classe {
public int maxFrequence(int[] nums, int k) {
// 1. Tri – O(n log n)
Tableaux.sort(nums);

long total = 0; // somme d'exécution à l'intérieur de la fenêtre
int gauche = 0; // pointeur gauche de la fenêtre coulissante
int res = 0; // meilleure taille de fenêtre vue jusqu'à présent

pour (int droite = 0; droite < nums.longueur; droite++) {
total += nombres[droit];

// Fenêtre de contrat alors qu'il a besoin de plus d'augmentations que nous (k)
pendant que ((long) nombres[droite] * (droite - gauche + 1) > total + k) {
total -= nombres[gauche];
gauche++;
}

// Mettre à jour la réponse
res = Math.max(res, droite - gauche + 1);
}
retour rés;
}
}
«» "

> ** Points clefs**
> * Utilisez `long` pour `total` et le produit pour éviter le débordement.
> * Le `temps` intérieur garantit que la fenêtre satisfait toujours la contrainte.

---

- Oui. Comment expliquer la logique de la fenêtre coulissante sur la mouche

> Après tri, toute fenêtre `[gauche...droite]` a une cible naturelle: l'élément le plus droit `nums[right]`. Tous les autres éléments de la fenêtre doivent être augmentés à cette cible.
> Le nombre d'accroissements requis équivaut à "cible * windowSize – total". Si cette valeur est ≤ k, la fenêtre est valide; autrement nous rétrécissons de la gauche jusqu'à ce qu'elle soit. (en milliers de dollars)

En expliquant, soulignez la distinction **locale contre globale** : la faisabilité de la fenêtre est une vérification *locale* ('k' est un budget *global*).
